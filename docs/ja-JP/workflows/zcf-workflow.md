# ZCF 6 段階ワークフロー

要件から振り返りまでを 6 段階に分けた標準フローです。AI へ段階ごとに指示することで漏れを減らします。

## 6 つのステージ

1. **Research**：現状把握、依存調査、リスク洗い出し  
2. **Design**：技術方針、API/DB/契約、テスト戦略  
3. **Implement**：タスク分割、具体的な手順とコマンド  
4. **Test**：テスト計画、ケース、優先度  
5. **Docs**：README/変更点/移行手順  
6. **Review**：今回の決定事項、残課題、次のアクション

## 使い方例

```
/zcf:workflow Build user login feature
```

必要に応じて追加文脈を付与：

- 依存ライブラリや API 仕様のリンク  
- 既存コードの抜粋  
- 非機能要件（性能/セキュリティ/可観測性など）

## ベストプラクティス

- 研究と設計をスキップしない。特に外部 API/権限/データモデルは最初に明確化する。  
- 実装ステージで作業単位を小さく分割し、`/git-worktree` を使って並列案を比較する。  
- Docs/Review を出力しておくと、次の会話や PR 説明に流用できる。
